# 디자인 문서: Kiro 한국 유저그룹 웹사이트

## 개요

Kiro 한국 유저그룹 웹사이트는 단일 페이지 애플리케이션(SPA)으로 설계되며, 메인 Kiro 웹사이트의 디자인 언어를 따릅니다. 핵심 특징은 애니메이션 유령 마스코트와 명확한 정보 계층 구조입니다. 기존 디자인 시스템을 활용하여 일관된 시각적 경험을 제공합니다.

## 아키텍처

### 기술 스택

- **마크업**: HTML5 (의미론적 구조)
- **스타일**: CSS3 (CSS 변수, 그리드, 플렉스박스)
- **스크립트**: Vanilla JavaScript (프레임워크 없음)
- **애니메이션**: CSS 애니메이션 + JavaScript 트리거
- **이미지**: SVG (마스코트), WebP/AVIF (배경)

### 페이지 구조

```
index.html
├── <head>
│   ├── 메타 태그 (viewport, charset, lang="ko")
│   ├── 폰트 로드 (Inter, Noto Sans KR)
│   ├── 디자인 시스템 CSS 변수
│   └── 스타일시트 (styles.css)
├── <body>
│   ├── Hero 섹션 (마스코트 + 제목)
│   ├── About 섹션 (유저그룹 소개)
│   ├── Features 섹션 (주요 기능/활동)
│   ├── Events 섹션 (예정된 이벤트)
│   ├── Join 섹션 (가입 방법)
│   ├── Contact 섹션 (연락처)
│   └── Footer
└── script.js (애니메이션 및 상호작용)
```

## 컴포넌트 및 인터페이스

### 1. Hero 섹션

**목적**: 사이트 방문자에게 즉각적인 임팩트 제공

**구성 요소**:
- Ghost_Mascot (애니메이션 이미지)
- 메인 제목 (64px, 800 weight)
- 부제목 (18px, 400 weight)
- CTA 버튼 (커뮤니티 가입 - Join 섹션으로 스크롤)

**레이아웃**:
- 데스크톱: 마스코트 좌측, 텍스트 우측 (2열 그리드)
- 모바일: 마스코트 상단, 텍스트 하단 (1열)
- 최소 높이: 100vh (뷰포트 높이)

**애니메이션**:
- 페이지 로드 시 마스코트: 0.6초 페이드인
- 마스코트: 8초 주기 누운 8자(∞) 플로팅 애니메이션 (평면 2D 이동)
- 텍스트: 0.8초 페이드인 (마스코트 이후 0.2초 딜레이)

### 2. About 섹션

**목적**: 유저그룹의 목적과 가치 전달

**구성 요소**:
- 섹션 제목 (48px, 800 weight)
- 설명 텍스트 (18px, 400 weight)
- 3개의 카드 (목적, 커뮤니티, 성장)

**카드 디자인**:
- 배경: rgba(255, 255, 255, 0.03) + 1px 테두리
- 패딩: 40px
- 테두리 반경: 12px
- 호버 상태: 배경 밝아짐, 위로 8px 이동, 그림자 추가

**레이아웃**:
- 데스크톱: 3열 그리드
- 태블릿: 2열 그리드
- 모바일: 1열

### 3. Features 섹션

**목적**: 유저그룹의 주요 활동과 혜택 소개

**구성 요소**:
- 섹션 제목
- 4-6개의 기능 카드 (아이콘 + 제목 + 설명)

**카드 디자인**:
- 아이콘: 48x48px, 보라색 그라데이션

- 설명: 16px, 400 weight
- 호버 시 아이콘 회전 (360도, 0.6초)

### 4. Events 섹션

**목적**: 지난 이벤트와 예정된 이벤트 정보 제공

**구성 요소**:
- 섹션 제목
- 이벤트 카드 목록 (지난 이벤트 1개 + 예정된 이벤트 최대 2개)
- 각 카드: 날짜 배지, 제목, 설명, 위치, 링크

**카드 디자인**:
- **레이아웃**: 좌측 날짜 배지 + 우측 이벤트 정보 (2열 그리드)
- **날짜 배지**: 
  - 예정된 이벤트: 보라색 그라데이션 배경
  - 지난 이벤트: 회색 그라데이션 배경
  - 연도/월/일 표시 (예: 2026 / Jan / 29)
- **지난 이벤트 표시**:
  - "종료된 이벤트" 배지 (우측 상단)
  - 카드 전체 불투명도 70% (흐릿하게 표시)
  - "자세히 보기" 링크 제거
  - 호버 시 약간의 상승 효과만 적용
- **예정된 이벤트**:
  - 호버 시 배경 밝아짐, 위로 4px 이동, 그림자 추가
  - "자세히 보기" 링크 표시

**이벤트 필터링 로직**:
- 현재 날짜 기준으로 자동 분류
- 지난 이벤트: 현재 날짜보다 이전 (최신순 정렬)
- 예정된 이벤트: 현재 날짜 이상 (오름차순 정렬)
- 표시: 지난 이벤트 1개 + 예정된 이벤트 최대 2개

**반응형 조정**:
- 태블릿(768px): 날짜 배지가 가로 레이아웃으로 변경
- 모바일(640px): 배지 크기 축소, 패딩 감소

## 애니메이션 및 트랜지션

### 주요 애니메이션

1. **페이드인 (Fade In)**
   - 지속 시간: 0.6초
   - 타이밍: ease-out
   - 용도: 페이지 로드 시 요소 진입

2. **플로팅 (Float)**
   - 지속 시간: 8초
   - 타이밍: ease-in-out
   - 용도: 마스코트 연속 애니메이션 (누운 8자 경로)
   - 경로: Lissajous 곡선으로 생성된 누운 8자(∞) 패턴
   - 이동 범위: ±100px 수평, ±50px 수직
   - 특징: 평면 2D 이동 (입체감 없음)

3. **호버 상태**
   - 지속 시간: 0.3초
   - 변화: 위로 2-8px 이동, 그림자 추가
   - 용도: 카드, 버튼 상호작용

4. **회전 (Rotate)**
   - 지속 시간: 0.6초
   - 각도: 360도
   - 용도: 기능 카드 아이콘 호버

### CSS 애니메이션 정의

```css
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

@keyframes float {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
}

@keyframes slideInUp {
  from { opacity: 0; transform: translateY(30px); }
  to { opacity: 1; transform: translateY(0); }
}
```

## 정정 속성 (Correctness Properties)

정정 속성은 시스템이 모든 유효한 실행에서 참이어야 하는 특성입니다. 이는 인간이 읽을 수 있는 명세와 기계가 검증 가능한 정확성 보장 사이의 다리 역할을 합니다.

### 속성 기반 테스트 개요

속성 기반 테스트(PBT)는 많은 생성된 입력에 대해 보편적 속성을 테스트하여 소프트웨어 정확성을 검증합니다. 각 속성은 모든 유효한 입력에 대해 성립해야 하는 형식적 명세입니다.

### 핵심 원칙

1. **보편적 정량화**: 모든 속성은 명시적인 "모든" 또는 "임의의" 문장을 포함해야 합니다
2. **요구사항 추적성**: 각 속성은 검증하는 요구사항을 참조해야 합니다
3. **실행 가능한 명세**: 속성은 자동화된 테스트로 구현 가능해야 합니다
4. **포괄적 커버리지**: 속성은 모든 테스트 가능한 수락 기준을 다루어야 합니다

## 정정 속성

### 속성 1: 마스코트 애니메이션 시작

**설명**: 페이지 로드 시 Ghost_Mascot은 항상 페이드인 애니메이션으로 시작되어야 합니다.

*임의의* 페이지 로드 이벤트에 대해, Ghost_Mascot 요소는 0.6초 이상의 페이드인 애니메이션을 가져야 합니다.

**검증: 요구사항 2.1**

### 속성 2: 마스코트 플로팅 애니메이션 연속성

**설명**: Ghost_Mascot이 표시된 후, 누운 8자(∞) 경로를 따라 평면으로 움직이는 플로팅 애니메이션은 중단 없이 반복되어야 합니다.

*임의의* 페이지 상태에서, Ghost_Mascot이 보이면 8초 주기의 누운 8자 플로팅 애니메이션이 실행 중이어야 하며, 3D 회전 효과 없이 순수 2D 평면 이동만 수행해야 합니다.

**검증: 요구사항 2.2**

### 속성 3: 반응형 레이아웃 일관성

**설명**: 모든 화면 크기에서 콘텐츠는 가독성 있게 표시되어야 합니다.

*임의의* 뷰포트 너비(640px, 768px, 968px, 1200px)에서, 모든 텍스트는 최소 14px 이상이어야 하고, 줄 길이는 80자 이하여야 합니다.

**검증: 요구사항 6.2**

### 속성 4: 색상 대비 준수

**설명**: 모든 텍스트와 배경의 색상 대비는 WCAG AA 기준을 충족해야 합니다.

*임의의* 텍스트 요소에 대해, 텍스트 색상과 배경 색상의 명도 대비는 최소 4.5:1 이상이어야 합니다.

**검증: 요구사항 8.1**

### 속성 5: 한국어 텍스트 렌더링

**설명**: 모든 한국어 텍스트는 'Noto Sans KR' 폰트로 올바르게 렌더링되어야 합니다.

*임의의* 한국어 텍스트 요소에 대해, 계산된 폰트 패밀리는 'Noto Sans KR'을 포함해야 합니다.

**검증: 요구사항 4.3**

### 속성 6: 페이지 로드 성능

**설명**: 페이지의 초기 콘텐츠는 3초 이내에 로드되어야 합니다.

*임의의* 표준 네트워크 연결에서, 페이지 로드부터 초기 콘텐츠 렌더링까지의 시간은 3000ms 이하여야 합니다.

**검증: 요구사항 7.1**

### 속성 7: 섹션 정보 흐름

**설명**: 페이지의 섹션들은 논리적인 순서로 배열되어야 합니다.

*임의의* 페이지 상태에서, 섹션의 순서는 다음과 같아야 합니다: Hero → About → Features → Events → Join → Contact → Footer.

**검증: 요구사항 3.2**

### 속성 8: 모션 감소 설정 존중

**설명**: 사용자가 모션 감소를 선호하면, 애니메이션은 비활성화되어야 합니다.

*임의의* 페이지 상태에서, `prefers-reduced-motion: reduce` 미디어 쿼리가 활성화되면, 모든 애니메이션의 지속 시간은 0.01ms 이하여야 합니다.

**검증: 요구사항 8.3**

## 구현 노트

### 파일 구조

```
project/
├── index.html
├── styles.css
├── script.js
├── events.json (이벤트 데이터)
├── kiro.png (마스코트 이미지)
└── assets/
    ├── images/
    │   ├── hero-bg.webp
    │   └── section-bg.webp
```

### 주요 구현 고려사항

1. **마스코트 이미지**: SVG 또는 PNG 형식, 최대 200KB
2. **CSS 변수**: 디자인 시스템의 모든 값을 CSS 변수로 정의
3. **JavaScript**: 프레임워크 없이 Vanilla JS 사용
4. **성능**: 초기 로드 시간 3초 이내 유지
5. **접근성**: WCAG 2.1 AA 기준 준수
6. **Events 섹션 구현**:
   - `events.json` 파일에서 이벤트 데이터 로드
   - 현재 날짜 기준으로 자동 분류 (지난 이벤트 vs 예정된 이벤트)
   - 지난 이벤트 1개 + 예정된 이벤트 최대 2개만 표시
   - 오류 처리: 데이터 로드 실패 시 사용자 친화적 메시지 표시
   - XSS 방지: HTML 특수문자 이스케이프 처리
